En esta sección, utilizamos el procesamiento de datos del terreno como ejemplo para describir el método de transformación de datos geodésicos. Dado que el servidor de Google Maps/Earth solo proporciona los datos del terreno en visualización gráfica, debemos obtener datos digitales del terreno de otras fuentes. Las bases de datos de datos del terreno de alta resolución (3" o más finos) como SRTM (Misión Topográfica Radar del Transbordador) o los datos DEM (Modelo de Elevación Digital) de USGS son necesarios. Además, dado que 3DWF se utiliza para modelar el flujo atmosférico a escala fina (metros hasta 100m), se necesita datos del terreno de alta resolución. En este proyecto, utilizamos el conjunto de datos de elevación del terreno de SRTM (Farr et al. 2007) con una resolución de 3 segundos de arco (~90m de resolución en el ecuador). Los datos cubren el área terrestre, casi global desde las latitudes 56S hasta 60N. Utilizamos la versión procesada del conjunto de datos SRTM versión 4 como se describe en Gamache (2005) en la que se rellenaron algunos de los agujeros de datos faltantes. Los datos originales están organizados en el sistema de coordenadas geodésicas WGS84 (Sistema Geodésico Mundial 84). Cuando los datos se aplican al modelo 3DWF, se transforman al sistema de coordenadas Este, Norte y Arriba (ENU) local (ver Fig. 3). Dado que 3DWF es un modelo de viento a escala fina y su dominio de modelo completo no está destinado a ser más grande que 20x20km, este sistema de coordenadas cartesianas es una buena elección con muy poca distorsión debido a la curvatura de la superficie terrestre.
