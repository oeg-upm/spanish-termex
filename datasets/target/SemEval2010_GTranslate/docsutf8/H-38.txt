Lyu. de informática e ingeniería La Universidad China de Hong Kong Shatin, NT, Hong Kong {Hxyang, King, Lyu}@cse.cuhk.edu.hk Resumen, mientras que el algoritmo de PageRank ha demostrado ser muy efectivo para las páginas web de clasificación, el rango, el rangoSe pueden manipular decenas de páginas web. Para manejar el problema de la manipulación y lanzar una nueva visión sobre la estructura web, proponemos un algoritmo de clasificación llamado DiffusionRank. DiffusionRank está motivado por los fenómenos de difusión de calor, que se pueden conectar a la clasificación web porque las actividades fluyen en la web se pueden imaginar como flujo de calor, el enlace de una página a otro puede tratarse como la tubería de un aire acondicionado yEl flujo de calor puede encarnar la estructura del gráfico web subyacente. Teóricamente, mostramos que DiffusionRank puede servir como una generalización de PageRank cuando el coeficiente de difusión de calor γ tiende al infinito. En tal caso 1/γ = 0, DiffusionRank (PageRank) tiene una baja capacidad de anti-manipulación. Cuando γ = 0, DiffusionRank obtiene la mayor capacidad de anti-manipulación, pero en tal caso, la estructura web se ignora por completo. En consecuencia, γ es un factor interesante que puede controlar el equilibrio entre la capacidad de preservar la web original y la capacidad de reducir el efecto de la manipulación. Se encuentra empíricamente que, cuando γ = 1, DiffusionRank tiene un efecto similar a la penicilina sobre la manipulación del enlace. Además, se puede emplear DiffusionRank para encontrar relaciones de grupo a grupo en la web, dividir el gráfico web en varias partes y encontrar comunidades de enlaces. Los resultados experimentales muestran que el algoritmo de DiffusionRank logra las ventajas mencionadas anteriormente como se esperaba. Introducción Si bien el algoritmo de PageRank [13] ha demostrado ser muy efectivo para clasificar las páginas web, los resultados inexactos de PageRank se inducen debido a las manipulaciones de la página web por personas para intereses comerciales. El problema de manipulación también se llama spam web, que se refiere a páginas hipervínculos en la red mundial que se crean con la intención de motores de búsqueda engañosos [7]. Se informa que aproximadamente el 70% de todas las páginas en el dominio .biz y aproximadamente el 35% de las páginas en el dominio .US pertenecen a la categoría SPAM [12]. La razón de la creciente cantidad de spam web se explica en [12]: algunos operadores de sitios web intentan influir en el posicionamiento de sus páginas dentro de los resultados de búsqueda debido a la gran fracción del tráfico web que se originan en las búsquedas y el valor monetario de alto potencial de esto.tráfico. Desde el punto de vista de los operadores del sitio web que desean aumentar el valor de clasificación de una página en particular para los motores de búsqueda, el relleno de palabras clave y el relleno de enlaces se están utilizando ampliamente [7, 12]. Desde el punto de vista de los administradores de motores de búsqueda, el spam web es muy perjudicial para las evaluaciones de los usuarios y, por lo tanto, su preferencia de elegir motores de búsqueda porque las personas creen que un buen motor de búsqueda no debe devolver resultados irrelevantes o de baja calidad. Se están empleando dos métodos para combatir el problema del spam web. Se emplean métodos de aprendizaje automático para manejar el relleno de palabras clave. Para aplicar con éxito los métodos de aprendizaje automático, necesitamos cavar algunas características textuales útiles para páginas web, para marcar parte de las páginas web como spam o no spam, luego para aplicar técnicas de aprendizaje supervisadas para marcar otras páginas. Por ejemplo, ver [5, 12]. Los métodos de análisis de enlaces también se emplean para manejar el problema de relleno de enlace. Un ejemplo es el Trustrank [7], un método basado en enlaces, en el que se utiliza la estructura del enlace para que las páginas de confianza etiquetadas por humanos puedan propagar sus puntajes de confianza a través de sus enlaces. Este documento se centra en el método basado en enlaces. El resto de los materiales se organizan de la siguiente manera. En la siguiente sección, damos una breve revisión de la literatura sobre diversas técnicas de clasificación relacionadas. Establecemos el modelo de difusión de calor (HDM) en varios casos en la Sección 3, y proponemos DiffusionRank en la Sección 4. En la Sección 5, describimos los conjuntos de datos en los que trabajamos y los resultados experimentales. Finalmente, sacamos conclusiones en la Sección 6. 2. Aunque el PageRank inverso no es perfecto en su capacidad de determinar la máxima cobertura, es atractivo debido a su tiempo de ejecución polinomial y su intuición razonable, en realidad invertimos el enlace original cuando intentamos construir el conjunto de semillas a partir de esas páginas que apuntan aMuchas páginas que a su vez apuntan a muchas páginas, etc. En el algoritmo, el gráfico aleatorio subyacente se establece como P = αb · A + (1 - αB) · 1 N · 1N × N, que es inducido por el gráfico web. Como resultado, r = −i + P. De hecho, la configuración más general para DiffusionRank es p = αB · A + (1 - αB) · 1 N · G · 1T. Según tal entorno, DiffusionRank es una generalización de Trustrank cuando γ tiende al infinito y cuando G se establece de la misma manera que Trustrank. Sin embargo, la segunda parte de Trustrank no es adoptada por nosotros. En nuestro modelo, G debería ser la verdadera teletransportación determinada por los usuarios de navegación de hábitos, distribución de popularidad en todas las páginas web, etc.;P debe ser el verdadero modelo de la naturaleza aleatoria de la red mundial. Establecer G de acuerdo con las páginas de confianza no será consistente con la idea básica de difusión de calor en un gráfico aleatorio. Simplemente establecemos G = 1 solo porque no podemos encontrarlo sin ningún conocimiento priori. Observación. En una interpretación de la red social, DiffusionRank reconoce por primera vez a un grupo de personas de confianza, que pueden no estar altamente clasificadas, pero conocen a muchas otras personas. Las personas inicialmente confiables están dotadas del poder de decidir en quién se puede confiar más, pero no pueden decidir los resultados finales de votación, por lo que no son dictadores.4.2 Ventajas A continuación, mostramos las cuatro ventajas para DiffusionRank.4.2.1 Dos formas cerradas primero, sus soluciones tienen dos formas, que son de forma cerrada. Conclusiones Concluimos que DiffusionRank es una generalización de PageRank, que es interesante en que el coeficiente de difusión de calor γ puede equilibrar la medida en que queremos modelar el gráfico web original y la medida en que queremos reducir el efecto de las manipulaciones de enlaces. Los resultados experimentales muestran que en realidad podemos lograr tal equilibrio estableciendo γ = 1, aunque la mejor configuración que incluye γi variable todavía está bajo mayor investigación. Esta característica anti-manipulación permite que DiffusionRank sea un candidato como penicilina para el spam web. Además, se puede emplear DiffusionRank para encontrar relaciones grupales de grupos y dividir el gráfico web en pequeñas comunidades. Todas estas ventajas se pueden lograr en la misma complejidad computacional que PageRank. Para la aplicación especial de anti-manipulación, DiffusionRank funciona mejor tanto en efectos de reducción como en su estabilidad entre los tres algoritmos.7.
