En esta sección, utilizamos el procesamiento de datos del terreno como ejemplo para describir el método de transformación de datos geodéticos.Dado que Google Maps/Earth Server solo proporciona los datos del terreno en la pantalla gráfica, tenemos que obtener datos digitales del terreno de otras fuentes.Son necesarios las bases de datos de terreno de resolución fina (3 ″ o más fino), como los datos SRTM (misión topográfica de radar de transbordador) o los datos de USGSS DEM (modelo de elevación digital).Además, dado que 3DWF se usa para modelar el flujo atmosférico a escala fina (metros de hasta 100 m), necesita datos de terreno de resolución fina.En este proyecto, utilizamos el conjunto de datos de elevación del terreno de SRTM (Farr et al. 2007) con la resolución de 3-Arcsegund (~ 90 m en el ecuador).Los datos cubren el área de tierra, casi global de 56 a 60n latitudes.Utilizamos el conjunto de datos SRTM de versión 4 procesado como se describe en Gamache (2005) en el que se llenaron algunos de los agujeros de datos faltantes.Los datos originales se organizan en el sistema de coordenadas geodéticas WGS84 (World Geodetic System 84).Cuando los datos se aplican al modelo 3DWF, se transforman en la coordenada local de este, norte y arriba (ENU) (ver Fig. 3).Dado que el 3DWF es un modelo de viento de escala fina y todo su dominio modelo no pretende ser mayor de 20 × 20 km, este sistema de coordenadas cartesianas es una buena opción con muy poca distorsión debido a la curvatura de la superficie de la Tierra.La transformación de los datos de WGS84 a la coordenada de ENU se realiza de la siguiente manera (Fukushima, 2006; Featherstone y Claessens, 2008).
